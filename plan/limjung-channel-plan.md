## 채널 콘셉트 한 문장

## ① 유튜브를 하는 이유

- 텍스트 자산(Threads·LinkedIn·책 2권)은 확보, **긴 체류로 신뢰를 만드는 매체는 공백**
- 벤치마크 결론: 5인 저자 중 본인 채널 보유는 조코딩뿐, 채널 = 출판·교육·창업 퍼널의 최상단
- 2026-05 시점 P3(6개월+)로 보류했던 항목 → 유튜브 챌린지(07-20 시작)로 실행

## ② 사업 파이프라인

- 출판: 〈n8n이 다 해줌〉 출간 · 〈클로드가 다 해줌〉 집필 중
- 강의·멘토링: 인프런, 기업·기관 출강, 부트캠프 멘토링 — 매주 "어디서 막히는지" 라이브 인풋 축적
- 콘텐츠·커뮤니티: Threads(@belle_epoque7) · LinkedIn · 12주 챌린지 운영 경험(시즌1 20명, 인증 142건)
- 구조: 무료(유튜브) → Threads·LinkedIn 팔로우 → 책 → 강의·멘토링 → 컨설팅. **유튜브 = 퍼널 최상단**

## ③ 주 목적 시청층 (페르소나)

- **[주] 지출은 제약되어 있지만, 성장 욕구로 공부하고 싶은 직장인**
  - ChatGPT 단발 사용까지는 도달, 반복 업무의 시스템화에서 막힘
  - 현상 유지 = 파편 튜토리얼 조합 → 중간에 막혀 포기 (강의 현장 반복 목격)
- [실험, 사이클 3+] 에이전틱 코딩이 막막한 주니어 개발자 — 지도상 별도 존(x≈82), 아크 1개로 반응만 실험

## ④ 유튜브 이후 유입 첫 단계 (CTA · 랜딩)

- 1단계: 영상 설명란 → **Threads·LinkedIn 팔로우** (뉴스레터는 아직 계획 없음, 추후 검토)
- 2단계: Threads·LinkedIn 게시물 → 책·인프런 강의 안내
- 표준 구조: 유튜브(무료) + 책·강의(유료) + 커뮤니티 삼각

## ⑤ 채널이 전달하려는 메시지

- "자동화는 개발자의 특권이 아니다 — 시행착오만 감수하면 누구나 자기 업무 시스템을 만들 수 있다"
- 페르소나 포지션: 5인 벤치마크(직장인 캐릭터·학자 권위·전문가 친절·형/실험러·비영리 교사형) 누구도 점유하지 않은 **시행착오형**

## 포지셔닝 (딥리서치 확정, 2026-07-19)

- 축: X = 타겟 기술 배경(비개발자 0 ↔ 개발자 100) · Y = 콘텐츠 산출 깊이(도구 소개 0 ↔ E2E 시스템 100)
- **확정 좌표: (30, 88)** — "비개발자·반기술 경계 대상 엔드투엔드 시스템 구축" 공백 존
- 공백의 이유 = 수요 부재가 아니라 **공급 난이도**(실운영+교수설계+제작 체력 동시 요구) → 셋 다 보유, 난이도가 곧 해자
- 밀집 존(툴 소개·단일 튜토리얼) 시청자의 "다음 단계" 길목 — 자연 유입 경로 존재

## 콘텐츠 포맷 (YC 오피스아워, 접근 C 확정)

- **빌드 인 퍼블릭**: 실제 운영 중인 시스템(업무일지 자동화·챌린지 봇·콘텐츠 파이프라인 등) 공개
- 매편 독립 완결 2부 구성: ①이 시스템이 실제로 돌아가는 모습 → ②**제약 버전으로 번역해 따라 만들기** (셀프호스팅 → 노는 스마트폰, 유료 API → 무료 티어)
